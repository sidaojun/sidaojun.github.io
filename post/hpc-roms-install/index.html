ROMS(Regional Ocean Modeling System)区域海洋模式是海洋研究机构常用的区域和大洋环流模式，由美国Rutgers University 和UCLA (University of California, Los Angeles) 合作开发。

ROMS在天河HPC系统上的编译

依赖软件环境

ROMS在x86_64平台上的编译需要依赖intel编译器、MPI编译器、Netcdf库和HDF5库。

依赖环境加载

在TH-HPC系统上部署有环境管理工具Modules，加载ROMS依赖的软件环境只需执行如下几行（可以写在~/.bashrc或者直接在当前终端执行）：

module add Intel_compiler/16.0.3
module add MPI/mvapich2-2.2/intel2016u3
module add netcdf/4.4-icc16-mvapich2

在系统中加载netcdf时，默认同时加载了HDF5.

准备编译所需的文件

1、创建目录

比方说要编译的问题是upwelling，可以创建目录upwelling

mkdir upwelling
cd upwelling

2、拷贝编译所需的文件

ROMS的编译需要对应问题的头文件、输入文件以及编译脚本build.bash，如upwelling问题需要upwelling.h和ocean_upwelling.in文件，拷贝这三个文件到当前目录：

cp /THL7/home/sidj/roms_source/ROMS/External/ocean_upwelling.in ./
cp /THL7/home/sidj/roms_source/ROMS/Include/upwelling.h ./
cp /THL7/home/sidj/roms_source/ROMS/Bin/build.bash ./

上面的roms_source为ROMS源码文件夹。

3、修改编译脚本build.bash

build.bash中要修改的内容包括：

  • ROMS_APPLICATION

修改为对应的问题，如UPWELLING

  export ROMS_APPLICATION=UPWELLING
  • MY_ROOT_DIR、MY_PROJECT_DIR、MY_ROMS_SRC、COMPLIERS

上面的几个变量用来定义相关文件所在的位置，在天河HPC系统上可以设置如下：

  export MY_ROOT_DIR=/THL7/home/sidj/roms_source
  export MY_PROJECT_DIR=${PWD}
  export MY_ROMS_SRC=${MY_ROOT_DIR}
  export COMPILERS=${MY_ROMS_SRC}/Compilers

其中MY_ROOT_DIR和MY_ROMS_SRC用来定义ROMS的源码位置，MY_PROJECT_DIR用来定义进行源码编译时所在的文件夹，COMPILERS用来定义包含.mk文件的路径。天河HPC系统可以按照上面命令进行设置。

  • 编译选项设置

对于天河HPC系统上的ROMS编译，要开启对应的并行编译开关，设置如下：

  #export       MY_CPP_FLAGS="-DAVERAGES"
  #export       MY_CPP_FLAGS="${MY_CPP_FLAGS} -DDEBUGGING"
  #export       MY_CPP_FLAGS="-D"

  export        USE_MPI=on                # distributed-memory parallelism
  export        USE_MPIF90=on             # compile with mpif90 script
  #export       which_MPI=mpich           # compile with MPICH library
  export        which_MPI=mpich2          # compile with MPICH2 library
  #export       which_MPI=openmpi         # compile with OpenMPI library

  #export       USE_OpenMP=on             # shared-memory parallelism

  export        FORT=ifort                #选择ifort
  #export       FORT=gfortran
  #export       FORT=pgi

  #export       USE_DEBUG=on              # use Fortran debugging flags
  export        USE_LARGE=on              # activate 64-bit compilation
  export        USE_NETCDF4=on            # compile with NetCDF-4 library
  export        USE_PARALLEL_IO=on        # Parallel I/O with Netcdf-4/HDF5

  #export       USE_MY_LIBS=on            # use my library paths below,注释掉此行
  • 其他变量设置

在build.bash脚本的最后，需设置以下变量：

  export        MY_HEADER_DIR=${MY_PROJECT_DIR}
  export        MY_ANALYTICAL_DIR=${MY_PROJECT_DIR}

  export        BINDIR=${MY_PROJECT_DIR}               #生成可执行二进制文件的存放目录
  export        SCRATCH_DIR=${MY_PROJECT_DIR}/Build    #编译时f90文件的存放目录

4、修改Linux-ifort.mk文件

此文件位于/THL7/home/sidj/roms_source/Compilers，即ROMS源码目录的下一级Compilers目录。

由于用到了netcdf和hdf5等的库，因此做以下修改（以HPC2系统为例）：

  # Library locations, can be overridden by environment variables.

  ifdef USE_NETCDF4
      NC_CONFIG ?= nc-config
      NETCDF_INCDIR ?= $(shell $(NC_CONFIG) --prefix)/include
      LIBS := -L/THL7/software/netcdf/4.4-icc16-mvapich2/lib -lnetcdf -lnetcdff \
            -L/THL7/software/hdf5/1.8.11-icc16-mvapich2/lib -lhdf5_hl -lhdf5 \
            -L/THL7/software/pnetcdf/1.8.0-icc16-mvapich2/lib -lpnetcdf -lz
  else
      NETCDF_INCDIR ?= /usr/local/include
      NETCDF_LIBDIR ?= /usr/local/lib
      LIBS := -L$(NETCDF_LIBDIR) -lnetcdf
  endif

编译

修改build.bash文件为可执行文件，然后编译

chmod +x build.bash  
./build.bash -j 8

编译成功后会生成oceanM文件。

ROMS在天河HPC系统上的运行

编辑运行脚本run.sh文件

run.sh脚本内容示例如下：

#!/bin/bash
yhrun -N 2 -n 56 -p TH_HPC2 ./oceanM ./ocean_upwelling.in > upwelling_log.out

n相当于mpirun命令中的np（MPI中并行的进程数），n应该等于输入.in文件中的NtileI与NtileJ的乘积。

提交运行脚本到天河计算节点

命令如下：

yhbatch -N 2 -p TH_HPC2 ./run.sh

yhrun 和yhbatch为天河超级计算机的作业提交命令，详见天河系统使用手册；-N为指定的计算节点数，-n为进程数，-p为用户可用的计算分区。
