Environment Modules 是一个用于简化shell初始化的工具，它能够让用户通过modulefiles动态的修改shell中的环境变量。

一、安装

必要条件

Modules的安装依赖tclsh，因此必须首先在系统中安装tcl（版本8.4以上）

安装步骤

按照通常的软件安装步骤进行，分为如下三步： - ./configure –prefix=/directory/you/want/to/install - make - make install

官方安装说明网址

https://modules.readthedocs.io/en/stable/INSTALL.html

二、module的配置和使用

初始化操作

为了每次登陆shell时，可以使用module，需要进行初始化操作，执行如下命令：

source /directory/you/want/to/install/init/bash

不同的shell类型执行对应脚本。

modulefiles路径的定义

module对环境变量的修改是通过操作modulefiles来实现的，因此需要对此进行定义。有以下几种方式： - 编辑init目录下的modulerc文件； - 编辑init目录下的.modulespath文件； - 在.bashrc中添加命令module use 文件名，类似:

module use /path/to/modulefiles

建议采用前面两种方式。

环境变量MODULEPATH保存了包含modulefiles的路径，即系统查找modulefiles的路径。

modulefiles文件的定位

系统会搜索MODULEPATH中的每个目录。MODULEPATH中的目录可以有很多层。如果用户加载的modulefile是个目录，系统会向下搜索真正的modulefile文件。

modufile默认版本的设置

可以通过设置要加载的modulefile的默认版本，来定义不知道具体版本时加载的modulefile文件。设置有两种方式： - 定义.version文件
如果我们在某个目录（如GCC）下，包含此软件的多个版本，那个可以在该目录下定义.version文件，实现默认版本的设置，如：

#%Module1.0
##
## The desired version of GCC
##
set ModuleVersion "4.9.4"

  • 定义.modulerc文件 方法如下：

    #%Module1.0
##
## The desired version of GCC
##
module-version "./4.9.4" default

    module的使用

以上就完成了module的配置工作，module的使用可以通过在命令行中输入module –help查看。常见的有module add、module rm、module swap等命令。

三、modulefile的编写

前面说到module对环境变量的修改需要操作modulefile，因此需要为要使用的应用软件编写对应的modulefile。modulefile的编写步骤如下：

modulefile文件的开头

一个modulefile以#%Module开头，后面跟版本号，当前的modulefile版本号是1.0，所以modulefile第一行为#%Module1.0。
注意：如果没有#%Module开头的一行，则moduelcmd.tcl不能够解析modulefile；如果版本号不存在，modulecmd.tcl假设modulefile和最新的版本兼容；版本号是有用的，因为modulefile格式可能会变。

modulefile中的命令

modulefile使用的是tcl命令，常用的命令有： - puts：puts命令用来输出字符串 - setenv：用来设置环境变量，它不会改变用户的环境变量。e.g. setenv prefix /install/directory

  • apend-path和prepend-path：用来在前面或者后面增加环境变量的值。e.g. prepend-path PATH $prefix/bin 注意：当向环境变量中增加值时，建议使用append-path，在必要的时候使用prepend-path，如想要覆盖系统中的默认软件版本。

  • prereq和conflict：它们用来控制modufile是否可以被加载。prereq命令用来定义当前的modufile被加载之前，哪些modulefiles必须先被加载。conflict命令用来定义哪些modulefiles和当前modulefile冲突。prereq和conflict后的变量可以是目录。

  • module-whatis：用来定义命令行中输入module whatis命令后响应的字符串。e.g. 在命令行中输入module whatis GCC/4.6.1

  • uname：uname命令提供系统信息的查找，如：uname sysname的结果为操作系统的名字

四、一个简单的case

#%Module1.0
##
## modules modulefile
##
## modulefiles/modules.  Generated from modules.in by configure.
##
proc ModulesHelp { } {
                global version prefix

                puts stderr "\tAdds gcc compiler to your environment variables."
                puts stderr "\n\tThis adds $prefix/* to several of the"
                puts stderr "\tenvironment variables."
                puts stderr "\n\tVersion $version\n"
}

module-whatis   "gcc/4.6.1"

# for Tcl script use only
set     version         gcc/gcc4.6.1
set     prefix          /vol-th/software/gcc-4.6.1
set     exec_prefix     ${prefix}
set     datarootdir     ${prefix}/share
set     INSTALL_DIR     ${prefix}

conflict                gcc
prepend-path            PATH    $prefix/bin
prepend-path            LD_LIBRARY_PATH $prefix/lib64
prepend-path            MANPATH $prefix/share/man
